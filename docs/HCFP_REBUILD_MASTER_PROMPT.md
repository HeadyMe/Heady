# HCFP Rebuild & Organization Master Prompt

## Executive Summary

I want to completely rebuild, clean, and reorganize all of my project files, repositories, documentation, and personal files across every drive, platform, and cloud environment. The goal is to achieve a system that is optimally structured, visually branded, up to date, and entirely free of unused, duplicate, or outdated materials.

This includes all work associated with both of my organizations—**HeadyConnection Inc.** (nonprofit) and **HeadySystems Inc.** (C‑Corp)—as well as personal archives, creative projects, and private records.

**CRITICAL DIRECTIVE**: Use image assets EXTENSIVELY and LIBERALLY across all UIs, folders, repositories, and documentation. Every platform, dashboard, workspace, and file directory should carry consistent visual representation, personality, and aesthetic continuity. Images should be used "very, very freely" as anchors of identity and familiarity.

---

## Phase 1: Global Audit & Discovery

### 1.1 Comprehensive Asset Mapping
- **Scan all environments**: Local drives (C:, F:, external SSDs), network storage, GitHub repos, Google Workspace, Dropbox, Notion, email attachments, and any external media
- **Create master inventory**: File names, paths, creation/modification dates, contributors, project relationships, sizes, and checksums
- **Categorize by entity**: HeadyConnection Inc., HeadySystems Inc., Personal/Erich
- **Tag everything**: `active`, `archived`, `deprecated`, `under_review`, `duplicate`, `personal`, `sensitive`

### 1.2 Repository & Project Analysis
- **GitHub audit**: All repos under HeadyMe, HeadySystems, HeadyConnection, sandbox, and personal accounts
- **Local project mapping**: heady-os/, HeadyAcademy/, notebooks/, docs/, scripts/, configs/
- **Colab/Notebook inventory**: All Google Colab, Jupyter notebooks across platforms
- **Notion workspace mapping**: All pages, databases, and connected documents

### 1.3 Media Asset Inventory
- **Locate all image assets**: Logos, icons, banners, Sacred Geometry patterns, product screenshots, concept art
- **Create central media library**: `Media/` directory with subcategories:
  - `Logos/` (primary, variants, formats)
  - `SacredGeometry/` (patterns, symbols, architectural elements)
  - `UI/` (icons, buttons, interface elements)
  - `Product/` (screenshots, mockups, demos)
  - `Brand/` (colors, fonts, style guides)
  - `Personal/` (photos, creative works, archives)

## Phase 2: Unified Structure & Taxonomy

### 2.1 Top-Level Architecture
```
HeadyEcosystem/
├── Organizations/
│   ├── HeadyConnection/           # Nonprofit operations
│   │   ├── Active/               # Current projects & operations
│   │   ├── Archive/              # Historical work & records
│   │   └── Media/                # Brand assets & visuals
│   └── HeadySystems/             # C-Corp operations
│       ├── Active/               # Core products & services
│       ├── Archive/              # Legacy code & research
│       └── Media/                # Technical assets & docs
├── Personal/
│   ├── Creative/                 # Art, writing, concepts
│   ├── Records/                  # Personal documents & archives
│   ├── Learning/                 # Research & development
│   └── Media/                    # Personal image collections
├── Shared/
│   ├── Media/                    # Central image repository
│   ├── Templates/                # Document & project templates
│   └── Tools/                    # Shared utilities & scripts
```

### 2.2 Naming Conventions
- **Repositories**: `Entity_ProjectName_Type_Version` (e.g., `HeadySystems_HeadyManager_Core_v3.1.0`)
- **Folders**: `Category_ProjectName_Status_YYYY` (e.g., `Active_HeadyOS_Docker_2025`)
- **Files**: `ProjectName_Component_Version.ext` (e.g., `HeadyManager_API_Documentation_v3.1.0.md`)
- **Images**: `Type_Entity_Purpose_Size.ext` (e.g., `Logo_HeadySystems_Primary_512px.png`)

### 2.3 Visual Branding Integration

- **Every dashboard**: Background themes, branded icons, Sacred Geometry accents

- **Every document**: Watermarks, headers, footers, visual separators
